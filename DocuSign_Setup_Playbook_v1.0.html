DocuSign Setup Playbook

Make the ACH form & Quote drop straight into DocuSign — set up once, reuse forever.

The idea in one line: the two forms now carry hidden anchor tokens. You build a DocuSign Template for each one time, point each field at its token, and from then on every send is New → Use a Template → pick form → enter customer email → send. No more dragging fields.
Why anchors and not just a saved template? The ACH payment schedule grows or shrinks with the number of payments, which pushes the signature block up and down the page. A field pinned to a fixed spot would drift onto the wrong line. A field tied to an anchor token follows that token wherever it lands — so it's always correct.

The anchor tokens (already built into the forms)

These are printed in white, ~6px — invisible on screen and on paper, but DocuSign reads them in the PDF text layer.

FormTokenDocuSign field to attach
ACH (v1.7)\s1\Signature
\n1\Name (printed)
\t1\Text field — Title
\d1\Date Signed
Quote (v1.2)\s1\Signature (customer)
\d1\Date Signed (customer)
\s2\Signature (PulmOne countersign — optional)

One time  Build the template (do this once per form)

  1. Open the form in your browser, fill nothing, and Print → Save as PDF (or use the form's PDF/send button) to get a clean blank PDF.
  2. In DocuSign: Templates → New Template → Upload that PDF. Name it "MiniBox ACH" (or "MiniBox Quote").
  3. Add a recipient role called Customer Signer (leave the email blank — it's a role, filled in at send time).
  4. Drag one field of each type onto the page (Signature, Name, Date Signed, Text). Exact spot doesn't matter yet.
  5. Click each field → in its properties open Anchor / "Place automatically" → type the matching token from the table above (e.g. \s1\). Set the offset so it sits just above the line.
  6. Save the template. Done — forever.

Tip: in field properties, turn off "All instances" if a token ever appears more than once; for these forms each token is unique.

Every time  Send a form (10 seconds)

  1. Fill the form in the toolkit as usual, then save/print it to PDF.
  2. DocuSign: Start → Use a Template → MiniBox ACH (or Quote).
  3. Replace the template's blank PDF with your filled PDF if you filled it outside DocuSign — fields re-snap to the anchors automatically.
  4. Enter the customer's email in the Customer Signer role → Send.

When you version a form up

Keep the same tokens in the new version (they're in the signature block). Re-upload the new PDF into the existing template — the fields re-snap to the tokens with zero re-tagging. That's the whole point: templates never break when you iterate.
